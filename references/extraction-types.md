# Extraction Types

> 7 种 HR 简历提取类型定义

## candidate - 候选人基本信息

**定义**: 简历中的个人基本信息，包括姓名、性别、年龄、联系方式等

**识别特征**:
- 简历头部区域的个人信息块
- 姓名通常在最顶部，独占一行
- 手机号 (11位数字，1开头)、邮箱 (含@符号)
- 年龄/出生日期、性别、所在城市
- 学历摘要 (本科/硕士等)

**简历示例**:
```
张三
男 | 28岁 | 北京
手机: 138-0000-0000 | 邮箱: zhangsan@email.com
本科 | 5年工作经验
```

**必填属性**: `name`
**可选属性**: `gender`, `age`, `birth_date`, `phone`, `email`, `city`, `education_level`, `years_of_experience`, `photo_url`

---

## experience - 工作经历

**定义**: 候选人的历史工作记录，包括公司、职位、时间段、职责描述

**识别特征**:
- "工作经历"、"工作经验"、"职业经历" 等标题下方的内容
- 时间段格式: `2020.07 - 至今`、`2018/03 - 2020/06`、`2018年3月-2020年6月`
- 公司名称 + 职位名称
- 以 `-` 或 `*` 开头的职责描述列表
- 嵌套的项目经历 (项目名 + 角色 + 描述)

**简历示例**:
```
2020.07 - 至今  ABC科技有限公司  高级后端工程师
- 负责核心交易系统的架构设计和开发
- 主导微服务改造项目，将单体应用拆分为12个微服务
- 项目: 智能风控平台 (2022.03-2022.12)
  角色: 技术负责人
  描述: 基于机器学习的实时风控系统，日处理交易500万笔
```

**必填属性**: `company`, `title`
**可选属性**: `period_start`, `period_end`, `duration_months`, `description`, `projects`

**projects 嵌套结构** (数组):
```json
{
  "name": "项目名称",
  "role": "担任角色",
  "period": "时间段",
  "description": "项目描述"
}
```

## education - 教育背景

**定义**: 候选人的学历信息，包括学校、专业、学位、时间段

**识别特征**:
- "教育背景"、"教育经历"、"学历" 等标题下方的内容
- 学校名称 (大学/学院/中学)
- 专业名称
- 学位关键词: 博士、硕士、本科、学士、大专、高中
- GPA/绩点信息
- 荣誉/奖学金

**简历示例**:
```
2016.09 - 2020.06  北京大学  计算机科学与技术  本科  GPA: 3.8/4.0
- 国家奖学金 (2018)
- 优秀毕业论文
```

**必填属性**: `school`, `degree`
**可选属性**: `major`, `period_start`, `period_end`, `gpa`, `honors`

**degree 枚举值**: `博士`, `硕士`, `本科`, `大专`, `高中`, `其他`

---

## skill - 技能标签

**定义**: 候选人掌握的技术技能、语言能力、软技能等

**识别特征**:
- "技能"、"专业技能"、"技术栈" 等标题下方的内容
- 技能名称 + 熟练度 (精通/熟练/了解/掌握)
- 技能名称 + 使用年限
- 编程语言、框架、工具名称
- 语言能力 (英语CET-6、日语N1等)
- 散布在工作经历中的技术关键词

**简历示例**:
```
- Python (精通, 8年)
- Java (熟练, 5年)
- Docker/K8s (熟练, 3年)
- MySQL/Redis (熟练)
- 英语 CET-6
```

**必填属性**: `name`
**可选属性**: `category`, `level`, `years`

**category 枚举值**: `语言` (编程语言), `框架`, `工具`, `数据库`, `外语`, `软技能`, `证书相关`, `其他`
**level 枚举值**: `精通`, `熟练`, `掌握`, `了解`, `其他`

---

## self_evaluation - 自我评价

**定义**: 候选人对自身能力、性格特点的主观描述

**识别特征**:
- "自我评价"、"个人简介"、"自我描述"、"个人总结" 等标题下方的内容
- 第一人称描述性文字
- 包含性格特质、能力优势、职业态度等关键词

**简历示例**:
```
8年后端开发经验，擅长高并发系统设计，有良好的团队协作能力和沟通能力。
热爱技术，保持持续学习的习惯，关注前沿技术动态。
```

**必填属性**: `text`
**可选属性**: `traits`

**traits**: AI 从原文中提取的关键特质标签数组，如 `["高并发系统设计", "团队协作", "沟通能力", "持续学习"]`

---

## job_intention - 求职意向

**定义**: 候选人的求职目标信息，包括期望职位、行业、薪资等

**识别特征**:
- "求职意向"、"期望职位"、"应聘岗位" 等标题或关键词
- 期望薪资范围 (25K-35K、面议等)
- 期望城市/工作地点
- 到岗时间 (一个月内、随时到岗等)
- 期望行业

**简历示例**:
```
求职意向: Python 高级开发工程师 | 期望薪资: 25K-35K | 到岗时间: 一个月内
期望城市: 北京/上海 | 期望行业: 互联网/金融科技
```

**必填属性**: `position`
**可选属性**: `industry`, `city`, `salary_min`, `salary_max`, `salary_unit`, `entry_date`

**salary_unit**: `K/月`, `万/年`, `面议`
**entry_date 常见值**: `随时到岗`, `一个月内`, `两周内`, `需商议`

---

## certification - 资质证书

**定义**: 候选人持有的专业证书、资格认证

**识别特征**:
- "证书"、"资质"、"认证"、"执照" 等标题下方的内容
- 证书名称 + 颁发机构
- 获得日期/有效期
- 证书编号
- PMP、CPA、AWS认证、软考等关键词

**简历示例**:
```
- AWS Solutions Architect Professional (2023.06, 有效期至2026.06)
- PMP 项目管理专业人士 (PMI, 2022.01)
- 软件设计师 (工信部, 2019.11)
```

**必填属性**: `name`
**可选属性**: `issuer`, `date`, `expiry`, `cert_id`
